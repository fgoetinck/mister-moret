## 📜 Code of Conduct

By participating in this project, you are expected to uphold standard professional behavior and treat all contributors with respect.

> [!TIP]
> Friendly and constructive communication is the key to a healthy open-source environment.

## ✨ How Can I Contribute?

### 🐛 Reporting Bugs

* Check the [issue tracker](https://github.com/fgoetinck/MisterMoret/issues) to see if the bug has already been reported.
* If not, create a new issue using the **Bug Report** template.
* Include as much detail as possible:
    * Steps to reproduce the issue.
    * Expected behavior vs. actual behavior.
    * Screenshots or code snippets if applicable.

### 💡 Suggesting Enhancements

* Check the [issue tracker](https://github.com/fgoetinck/MisterMoret/issues) to see if the enhancement has already been suggested.
* If not, create a new issue using the **Feature Request** template.

## 🛠️ Pull Requests

1. **Fork** the repository.
2. **Create a new branch** for your feature or bug fix:
   ```bash
   git checkout -b feature/your-feature-name
   # or
   git checkout -b fix/your-fix-name
   ```
3. **Make your changes** following the existing code style.
4. **Commit your changes**:
   ```bash
   git commit -m 'Add some feature'
   ```
5. **Push to the branch**:
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Open a Pull Request** against the `dev` branch.

## 🚀 Development Setup

To get started with development, you'll need:

* **.NET 8.0 SDK** or later.
* Your favorite IDE: **Visual Studio**, **JetBrains Rider**, or **VS Code**.

> [!NOTE]
> Ensure all tests pass before submitting your Pull Request.
