# Contributing to MisterMoret

First off, thank you for considering contributing to MisterMoret! It's people like you that make MisterMoret such a great tool.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct. (Link to your CoC if you have one, or just mention standard professional behavior).

## How Can I Contribute?

### Reporting Bugs

* Check the [issue tracker](https://github.com/fgoetinck/MisterMoret/issues) to see if the bug has already been reported.
* If not, create a new issue using the **Bug Report** template.
* Include as much detail as possible, including steps to reproduce, expected behavior, and actual behavior.

### Suggesting Enhancements

* Check the [issue tracker](https://github.com/fgoetinck/MisterMoret/issues) to see if the enhancement has already been suggested.
* If not, create a new issue using the **Feature Request** template.

### Pull Requests

1. Fork the repository.
2. Create a new branch for your feature or bug fix: `git checkout -b feature/your-feature-name` or `git checkout -b fix/your-fix-name`.
3. Make your changes.
4. Ensure your code follows the existing style.
5. Commit your changes: `git commit -m 'Add some feature'`.
6. Push to the branch: `git push origin feature/your-feature-name`.
7. Open a Pull Request against the `dev` branch.

## Development Setup

* Requires .NET 8.0 SDK or later.
* The solution can be opened in Visual Studio, JetBrains Rider, or VS Code.
